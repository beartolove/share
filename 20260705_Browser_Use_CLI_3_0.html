Browser Use CLI 3.0

建立時間：2026-07-05 17:32:16（Asia/Taipei）引用來源：https://www.facebook.com/share/1EMoaKg2qe/

重點摘要

  • Browser Use CLI 3.0 主打透過 Chrome DevTools Protocol 操控瀏覽器，減少傳統 click/type/screenshot 猜測式操作。
  • 貼文指出它具備「網域技能」設計，可保存特定網站的登入流程、選擇器與特殊情境處理方式，方便下次自動套用。
  • 若操作過程缺少指令，Agent 可自行補寫所需程式碼接續執行，降低流程卡住的機率。
  • 支援本機 Chrome、雲端與自訂端點三種環境；安裝建議用 Python 3.12 搭配 uv 升級 browser-use。

提醒：本文為分享內容整理，涉及工具功能、安裝方式與使用體驗，實際設定仍建議以官方文件與原始 repo 為準。

https://www.facebook.com/share/1EMoaKg2qe/

想裝 Browser Use CLI 3.0？在你的 AI Agent裡貼這段：

用 Python 3.12，透過 uv 安裝或升級 browser-use 到最新穩定版。安裝完成後，從 browser-use skill 註冊這個 skill，然後連接到我的瀏覽器。如果設定或連線有問題，照這個 repo 的說明處理：https://github.com/browser-use/browser-use

最後去官網申請一組免費 API Key 就能開始用了。

Browser Use CLI 3.0，103k star 的瀏覽器自動化神器。

之前用 Coding Agent 操控瀏覽器，大多是靠 click()、type() 和 screenshot 在「猜」。每次操作都要把整個 DOM 樹塞進上下文，又慢又吃 token。

Browser Use CLI 3.0 換了一個思路。它直接走 Chrome DevTools Protocol，繞過那些猜測式的操作，讓 Agent 跟瀏覽器之間的溝通變得直接很多。

最讓我驚豔的是「網域技能」這個設計。你在某個網站摸索出來的登入流程、元素選擇器、特殊狀況的處理方式，全部會被存成技能。下次碰到類似的網站，Agent 自動套用，不用從頭學。

另一個細節是：如果執行中碰到缺指令的情況——比如要上傳檔案卻沒有現成函式——Agent 會自己寫出需要的程式碼接著跑，不會卡死在那邊。

它支援三種環境：本機 Chrome、雲端、自訂端點。安裝也簡單，在 Claude Code 或 Codex 裡貼一段指令就搞定，API Key 免費申請。

一個人就是一間公司，能自動化的就不要手動。Browser Use 把瀏覽器操作這塊的門檻砍到很低，值得裝起來試。
