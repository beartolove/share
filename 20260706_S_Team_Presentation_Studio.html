S-Team 簡報工作室

建立時間：2026-07-06 00:21:10（Asia/Taipei）引用來源：https://www.facebook.com/share/p/1EfDjWEK9f/

重點摘要

  • 貼文分享 S-Team 簡報工作室這套 AI 簡報製作流程，可依需求產出簡報與講稿。
  • 常用輸出格式包含簡報 HTML / PPTX，以及講稿 HTML / PDF / DOCX；作者偏好 HTML 簡報，方便跨裝置開啟。
  • 製作方式可從零訪談、依既有規劃製作，或由使用者直接規劃每頁內容後交給簡報團視覺化。
  • 目前內建 tech-mystery、minimal-modern、editorial、bauhaus 四種風格，也可用 /style-create 新增 design token。
  • 作者提醒內容仍建議自行檢查，但製作團隊可先協助完成初步考證與整理。

提醒：本文為分享內容整理，涉及開源工具、工作流程與個人使用經驗，實際使用仍建議以原始 repo 與自身需求驗證。

原文

https://www.facebook.com/share/p/1EfDjWEK9f/

常常看很很多人分享自己用AI做Slide 原本以為這是很基礎的用法

但後來還是經常看到許多人在詢問相關的實際製程 讓我覺得AI簡報製作 這件事 似乎還是有一段GAP 所以心血來潮分想一下自己常常使用的製程團隊

S-Team 簡報工作室 https://github.com/chemistrywow31/S-Team

可依照需求製作完整的簡報以及講稿 看實際跟工作室提出的需求是什麼

簡報常用格式 HTML, PPTX 講稿常用格式 HTML, PDF, DOCX (但我自己用其實講稿我都掃過去而已不會完全照著說)

目前簡報我個人比較偏愛使用HTML 簡單起個web server 到哪都能開 不用綁定windows office

現在基本上我有公司內部sharing, 專案提案, 面試自我介紹 跟其他所有需要簡報的場合 我都是用這個製程去產出簡報了 最後再依照自己的實際需求去微調

製作團隊也會去考證內容的正確性 建議最後自己還是要看過 能夠省去初步繁雜的考證已經幫助很大

簡報可以由幾種方式製作 1. 空手而來 從零開始訪談 2. 已經由其他地方產生簡報規劃需求 3. 直接規劃好每一頁的內容 簡報團負責視覺化

按照自己的需求彈性使用

簡報目前內建四種風格 1. tech-mystery ・ 科技神秘風 2. minimal-modern ・ 簡約現代風 3. editorial ・ 編輯風 4. bauhaus ・ 包浩斯風 風格不是固定 可使用 /style-create 自己新增喜歡的design token

另外因為claude code內建的system prompt有很多軟體工程用語 實際製作簡報建議使用cowork

這幾份就是利用S-Team readme.md 請簡報製作團 用口語化方式製作一份自我介紹簡報 人類0介入
