重點摘要

  • Andrew Ng 將 0-to-1 AI 產品開發拆成三個回饋迴圈：Agentic Coding Loop、Developer Feedback Loop、External Feedback Loop。
  • 三個迴圈對應不同時間尺度：分鐘級由 Agent 寫 code 與測試；小時級由 Developer 調整產品方向與 spec；天／週級由 Users 驗證產品是否做對。
  • 核心觀點不是只讓 Agent 跑得快，而是讓人類持續把「脈絡優勢」注入系統，決定產品到底該往哪裡跑。
  • Andrew Ng 的框架偏產品開發視角；LangChain 的 Loop Engineering 偏系統架構視角，兩者可以疊加使用。
  • 隨著 coding agent 加速開發，工程師、PM、設計師的邊界正在模糊；關鍵能力變成在 build 與 get feedback 之間取得平衡。

Andrew Ng（DeepLearning.AI 創辦人）最新一期 The Batch newsletter 聊了他做 0-to-1 產品時用的三個迴圈。跟我們之前介紹的 LangChain「Loop Engineering」是同一個方向，但 Andrew Ng 的切法更偏產品開發者的視角。

發布日期：2026/6/26

Key Points： • 「Loop engineering」已經成為 AI 圈的熱門術語（被 Boris Cherny 跟 Peter Steinberger 帶紅） • Andrew Ng 提出他自己的三個迴圈：Agentic Coding Loop、Developer Feedback Loop、External Feedback Loop • 三個迴圈的時間尺度不同：分鐘級 → 小時級 → 天/週級 • 核心觀點：人類的價值不是「品味」，是「context advantage」

▋ Loop 1：Agentic Coding Loop（分鐘級）

''' Product Spec + Evals → Coding Agent 寫 code → 測試 → 不 OK → 繼續改 → OK → 產出可用的版本 '''

Agent 自己寫、自己測、自己改，直到 bug-free 且符合 spec。每幾分鐘就能跑出一個新版本。

Andrew Ng 的實例：他幫女兒做一個打字練習 App，coding agent 可以自己跑一個小時，用瀏覽器自己測自己改，完全不需要他介入。

▋ Loop 2：Developer Feedback Loop（小時級）

''' 開發者看目前的產品 → 給高層次的方向（功能、UI、流程） → 更新 spec → 回到 Loop 1 '''

去年開發者還在幫 agent 做 QA（手動找 bug）。但現在 agent 自己測試能力大幅提升，開發者的角色從「抓 bug」升級到「做產品決策」——要什麼功能、UI 怎麼改、用戶流程怎麼設計。

打字 App 的例子：他改了好幾次視覺設計、女兒可以解鎖什麼貓咪衣服（她很愛貓）、大人登入後怎麼引導小孩的學習體驗。

Andrew Ng 特別提到一個觀點——很多人說人類的貢獻是「taste（品味）」，但他更喜歡用「context advantage（脈絡優勢）」來描述：

> 人類比 AI 更了解使用者和產品需要運作的環境。只要人類知道 AI 不知道的東西，human-in-the-loop 就是必要的，因為需要把那些知識注入系統。

▋ Loop 3：External Feedback Loop（天/週級）

''' 找朋友試用 / Alpha 測試 / A/B testing → 收集數據跟反饋 → 更新開發者的 product vision → 回到 Loop 2 '''

最慢的一層——通常需要幾小時到幾週。但這層決定了你到底在做對的東西還是錯的東西。

▋ 三層迴圈的對照

''' 時間尺度 誰在跑 做什麼 Loop 1 分鐘 Agent 寫 code + 測試 Loop 2 小時 Developer 產品方向 + spec 調整 Loop 3 天/週 Users 驗證你做的東西對不對 '''

▋ 跟我們之前聊的 Loop Engineering 的比較

LangChain 那篇講的四層迴圈（Agent → Verification → Event-Driven → Hill Climbing）偏「系統架構」視角——怎麼讓 agent 自動化、穩定、自我改進。

Andrew Ng 的三層迴圈偏「產品開發」視角——不只是讓 agent 跑得好，而是怎麼決定 agent 該跑什麼方向。

兩者不衝突，可以疊在一起：LangChain 的 Loop 1-2 大約對應 Andrew Ng 的 Loop 1，LangChain 的 Loop 4（Hill Climbing）是自動化版的 Loop 2 跟 3。

💡 Andrew Ng 最後提到一個趨勢我覺得很值得關注：

隨著 coding agent 加速軟體開發，越來越多工程師開始扮演「部分 PM 的角色」。同時 PM 跟設計師也在做更多工程的事。職能邊界正在模糊——跟之前 Boris Cherny 講的五種 archetype 是同一個趨勢。

對很多工程師來說，最難的轉變不是技術，是學會在「build（把 vision 變成 spec）」跟「get feedback（用反饋更新 vision）」之間找到平衡。
