重點摘要

  • Gemma 4 這次更新主軸放在效能、穩定度與使用體驗，尤其針對推論速度、Agent 工具呼叫與視覺任務做改善。
  • 在 NVIDIA Hopper 架構 GPU（H100、H200）上支援 Flash Attention 4，貼文指出 prefill 吞吐量提升 25%～70%，TTFT 最多縮短 31%。
  • Agent 相關修正包含 chat template 與 tool calling：對話格式更順、工具呼叫格式與執行行為更可預期。
  • 視覺任務提供更可調的選項：預設 vision bucket 仍偏 token 效率，但高解析細節任務可將 max_soft_tokens 調到 1120。
  • 貼文也提到模型「偷懶」問題改善：較少回答到一半就收尾或刻意保留，回答完整度提升。

原本以為還要等一年才有更厲害的開源模型，沒想到 Gemma 4 就直接進化了！

Gemma 4 這次的更新，重點放在效能、穩定度與使用體驗。

速度上最有感的是 Flash Attention。這次在 NVIDIA Hopper 架構的 GPU 上 （也就是 H100、H200），全面支援 Flash Attention 4， prefill 吞吐量提升 25% 到 70%，首字回應時間（TTFT）最多可以縮短 31%。

模型針對 Agent 呼叫修了兩個常被抱怨的地方。一是 chat template，將對話格式的處理變得更順，不會再出現奇怪的拼接問題。二是 tool calling，先前工具呼叫時偶爾會格式不一致或執行失敗，這次做了修補，行為變得比較可預期。

視覺提供可調選項，以往 Gemma 4 預設的 vision bucket 是 280，這是為了 token 效率所做的取捨。但如果你的任務需要處理高解析度視覺細節，可以手動把 max_soft_tokens 調到 1120。

最後一項是「偷懶」問題。先前模型會回答到一半就收尾，或是刻意保留不講完，這次大幅減少了這類狀況，回答的完整度提升。
