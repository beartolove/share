AI Agent 工作原理與 Harness 工程：一個動畫徹底搞懂

建立時間：2026-07-19 09:56:52（Asia/Taipei）
引用來源：YouTube 原始影片

重點摘要

  • Agent 不是單次模型呼叫，而是模型與工具的循環系統。
  • ReAct 可拆成思考、行動、觀察，反覆直到結果可用。
  • Plan-and-Execute 會先規劃任務清單，再逐步執行。
  • 大模型沒有真正記憶，需要設計上下文管理。
  • 多 Agent 協作可把複雜任務拆給不同角色處理。
  • Harness 是產品化 Agent 的外層防護網。

一句話總結

AI Agent 的核心不是神祕魔法，而是讓模型在工具呼叫、觀察結果、修正策略之間循環；要讓它穩定可用，就必須用 Harness 工程把風險包起來。

核心概念表

工具呼叫

讓 AI 不只回答文字，而能呼叫搜尋、渲染、檢查、資料查詢等外部能力。

ReAct

Reasoning + Acting：先想、再做、看結果，必要時再想再做。

Plan-and-Execute

先列計畫，再依步驟執行，適合較長或較複雜的任務。

上下文管理

控制哪些歷史、工具結果與限制要放進模型上下文。

多 Agent 協作

一個 Agent 拆任務，其他子 Agent 分工處理，降低單一上下文負擔。

Harness 工程

輸入過濾、格式清洗、參數校驗、錯誤重試、輸出檢查等外層防護。

Logo 動畫 Agent 的工作流程

  1. 1. 使用者提出任務：例如要求生成蘋果 Logo 的 SVG 動畫。
  2. 2. AI 判斷需求：模型發現需要 Logo 素材，不應憑空硬畫。
  3. 3. 呼叫工具：外層系統執行素材搜尋函式，取得 SVG。
  4. 4. 觀察結果：AI 讀取工具回傳內容，繼續生成動畫。
  5. 5. 自我檢查：渲染 SVG 成圖片，用視覺檢查工具判斷是否符合需求。
  6. 6. Harness 防護：在輸入、工具參數、輸出與錯誤處理上加上工程防線。
