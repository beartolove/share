Agent Memory 分類與工程治理：短期、長期與可治理閉環

建立時間：2026-07-19 10:13:32（Asia/Taipei）
引用來源：YouTube 原始影片

重點摘要

  • Agent Memory 不是簡單的長短二分，而是多種載體的組合。
  • 短期記憶是 prompt 內的 working memory；長期記憶在外部儲存。
  • 向量資料庫、知識圖譜、結構化資料庫、事件流各自負責不同召回問題。
  • 長期記憶應寫入穩定偏好、使用者畫像與專案事實。
  • 一次性對話、臨時變數與敏感資料不應落盤。
  • 生產環境要設計命名空間隔離、脫敏加密與 Memory GC。

一句話總結

Agent Memory 的核心不是把所有對話都存起來，而是判斷什麼該寫入、如何召回、何時刪除，並讓它成為可治理的工程閉環。

四種長期記憶載體

向量資料庫

語意模糊召回，例如根據問題找出相近偏好或歷史脈絡。

知識圖譜

實體關係推理，例如人、專案、任務、資源之間的關聯。

結構化資料庫

精確狀態查詢，例如專案狀態、欄位化偏好、帳號設定。

事件流

時序與因果回溯，例如使用者何時改變偏好、任務如何演進。

短期記憶

當前上下文中的草稿紙，速度快但不持久。

長期記憶治理

定期合併、剔除、脫敏、隔離，避免記憶庫變髒。

寫入判斷表

類型例子判斷原則
應進長期使用者畫像、專案事實、穩定偏好未來會重複使用，且具有穩定性
