給非技術人員的 GitHub 教學：Vibe Coding 必學基礎技能

建立時間：2026-07-19 09:39:34（Asia/Taipei）
引用來源：YouTube 原始影片

重點摘要

  • GitHub 可以理解成程式碼專用的雲端硬碟。
  • Git 是本機版本管理工具，負責追蹤與回復修改。
  • commit 是安全存檔點，push 是把進度送上 GitHub。
  • clone / pull 是取得專案與同步最新進度。
  • branch / PR / merge 是安全開發、審查與合併的流程。
  • conflict 可由 AI 協助，但人要決定產品取捨。

一句話總結

Vibe Coding 不代表完全不用懂開發流程；至少要理解 Git 與 GitHub 的基本語言，才能讓 AI 幫你寫程式時更安全、更可控。

核心術語白話表

GitHub

雲端保存、分享與協作程式碼的平台。

Git

在本機追蹤修改紀錄，讓程式碼可回復。

commit

完成一段穩定進度後的安全檢查點。

push / pull

push 上傳本機進度；pull 拉回雲端最新版本。

branch

開新功能時的平行工作區，用來隔離風險。

PR / merge

PR 是改動提案；merge 是審查後合併回主線。

AI Coding 的安全協作流程

  1. 1. clone：第一次加入專案，把 GitHub 上的專案正式抓到本機。
  2. 2. branch：開發新功能前先開分支，不直接動穩定的 main。
  3. 3. commit：每完成一段可用進度就存檔，留下可回復檢查點。
  4. 4. push：把本機分支推到 GitHub。
  5. 5. PR：請 AI 整理改動說明與測試方式，送出審查。
  6. 6. merge + pull：確認後合併回 main，並記得本機 pull 最新版本。
