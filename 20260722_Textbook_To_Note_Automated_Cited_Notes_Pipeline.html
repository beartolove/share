重點摘要

  • 這是一套把合法取得的 PDF / EPUB 教科書轉成 AI 可搜尋知識庫與結構化筆記的開源流程，主打 local-first、token-frugal 與每條主張可追溯出處。
  • 它解決的不是「叫 AI 幫我摘要一本書」而已，而是跨 40 多本指定教科書時，如何可靠搜尋、加權來源、避免幻覺、產出可學習的高密度筆記。
  • 流程分成五關：PDF/OCR 轉換、語意切塊搜尋、跨書來源加權、盲寫加模板的筆記產生、教科書圖片抽取與 audit。
  • Repo 直接放 AGENTS.md 與五套中英雙語筆記模板，使用方式偏向「把 repo 交給 Claude Code / coding agent 讀 AGENTS.md 安裝」。
  • 作者也提醒：OpenEvidence 這類外部實證來源會引用出錯，必須做 verify；工具本身不含書籍內容，需使用自己合法取得的書。

提醒：這套工具不含任何書籍內容；若要使用，需以自己合法取得或開放授權的教科書建立知識庫。

五階段流程

階段做什麼解決什麼問題
1. ConvertPDF / EPUB 轉 clean Markdown；掃描頁走 OCR ladder，壞字型與雙欄排版都要檢查先把書變成 AI 能穩定讀的格式，避免看似有文字其實亂碼或漏頁
2. Chunk依標題與脈絡切成可語意搜尋的單位避免固定字數切塊造成語意斷裂或內容稀釋
3. Search / Weight用本地 LanceDB 與 embeddings 跨書搜尋，並依指定用書、官方教科書與年份加權在書海中優先找到最可信、最該引用的來源
4. Note Template用盲寫＋模板產生結構化筆記，每條主張都要有出處把 AI 從自由發揮拉回有根據、有格式、可查證的輸出
5. Figures / Audit抽取圖表、圖說與版面，並用決定論 audit 檢查教科書重點常在圖、表、演算法與分類圖；不能只抽文字

為什麼值得注意

這個專案的價值不只是自動化摘要，而是把「可信來源、語意搜尋、結構化筆記、引用查核、圖片抽取」串成一個可反覆使用的學習生產線。對專科考試、研究入門、跨書整理、企業內訓或任何需要消化大量高品質文件的場景，都很有啟發。

它也很符合 AI agent 的新使用方式：repo 裡提供直接寫給 AI 的 AGENTS.md，讓使用者把安裝部署工作交給 Claude Code 或其他 coding agent。

使用與風險提醒

  • 資料合法性：工具不含書籍內容，必須使用自己合法取得或開放授權的教科書。
  • 幻覺與引用：筆記不能只靠模型記憶生成，每條主張應能回到來源。
  • OCR / 版面錯誤：掃描頁、雙欄排版、壞字型與圖片抽取都可能出錯，需要檢查機制。
  • 外部證據源：OpenEvidence 等臨床實證工具可能引用錯置，必須 verify。

《跨教科書 自動筆記生產流程：專科考試下的產物》

四十多本教科書，怎麼可能讀得完？

我從醫學生時代就很愛做筆記，這些年累積了幾千份。但老實說，在資訊取得這麼容易、要學的東西又這麼多的現在，已經很難再把每一份都按自己的架構做到相同品質了。

後來想通一件事：資訊越多，能信任的高品質來源反而越珍貴，而教科書就是最好的來源之一。可是專科指定參考書就四十多本，同個概念還散在好幾本書的不同章節裡，要真的都念過幾乎不可能。

慢慢調教教出 AI 自動筆記系統

很幸運這年代有了 AI，這種長上下文正是他擅長的，可是一次丟他幾百本書也不切實際。所以我的做法是搭配好的內容搜尋跟資料庫管理，再把我做筆記的流程講清楚、調教給 Claude，在避免幻覺的前提下讓他有根據、有架構地代勞整理，我專心把整理好的高密度資料吸收進腦袋就好。

於是這變成一個大型的教科書 LLM Wiki 專案，一路過了五關：把書轉成 AI 讀得動的格式（掃描頁OCR、亂碼檢驗、雙欄排版都是坑）、把內容切成有脈絡的搜尋單位搭配語意搜尋、在書海加權找到最該信任的來源、設計盲寫加 template 的筆記流程讓每條主張都有出處、最難的是從書裡抽圖片，每本書邏輯都不一樣，還要有省 token 的 audit 機制去把關🤣

開源整套方案

今天把整套開源了！repo 裡放了一份 AGENTS.md，直接寫給 AI 看的部署說明。因為會來用的人八成也帶著自己的 AI，所以用法就是把 repo 丟給 Claude Code 說「讀 AGENTS.md 幫我裝起來」，剩下讓他自己來😆 另外我也把每天在用的五套筆記模板（中英雙語）直接附在 repo 裡，可以照著改成自己的。

順便加碼：這套筆記流程還能往外接臨床實證來源，我自己接的是 OpenEvidence。這一組提問加 verify 的工具，也另外開源一個小專案 openevidence-tools。這個要特別謝謝 林協霆醫師：我原本自己寫自動化去接 OE 五月時被擋掉，看到他的作品才找到出路，而且後來讓查核真正成立的關鍵概念幾乎都來自他，包括 transitive citation（綜述引用別人的數字、OE 卻算在綜述頭上）這個失效模式的命名跟整套查核架構。提醒一下，OpenEvidence 是會引用出錯的，一定要自己做 verify！

後記：現代學習被極限加速了

做完最大的感想是，對一個專業主題取得快速、深入、教科書等級的理解，變成按下去就會發生的事。學習變得前所未有的容易，早幾年真的不敢想像。這套不限定復健科，甚至不限定醫學，希望能幫助到所有需要學習的人～
